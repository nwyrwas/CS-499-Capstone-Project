üß© Original Artifact ‚Äì CS-340 Client-Server Development

The starting artifact originated from the CS-340 Client-Server Development course. That project implemented a Flask-based CRUD web service connected to MongoDB, enabling basic operations for animal data management. While functional, the original version had several limitations: it lacked real-time visualization, contained minimal security controls, and had limited scalability. The design was primarily procedural and not yet modularized for large-scale extension.

These constraints made it an excellent foundation for enhancement. The goal was to transform a working but rudimentary prototype into a sophisticated analytical environment with secure coding practices, strong visual presentation, and interactive elements suitable for non-technical users such as shelter staff or data analysts.

üöÄ Enhanced Artifact ‚Äì Capstone Dashboard

The enhanced dashboard introduces significant architectural, functional, and design improvements. By integrating Dash, I restructured the application into a component-based model that reacts to user events through callback functions and dynamic rendering. Matplotlib powers graphical output, while Dash Leaflet provides geographic mapping of animal rescue data.

Security and performance were central enhancements. I implemented parameterized MongoDB queries to mitigate injection vulnerabilities, optimized aggregation pipelines to reduce query time, and separated application logic into distinct modules for maintainability. The user interface now features a dark theme that emphasizes accessibility, readability, and professionalism.
